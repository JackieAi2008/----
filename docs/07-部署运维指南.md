## 二、域名配置

### 2.1 DuckDNS配置（免费）

```bash
# 1. 访问 https://www.duckdns.org
# 2. 使用GitHub/Google/Reddit账号登录
# 3. 创建子域名，如：cimc-calendar
# 4. 记录生成的token

# 5. 创建自动更新脚本
mkdir -p ~/duckdns
cd ~/duckdns

cat > duck.sh << 'EOF'
echo url="https://www.duckdns.org/update?domains=cimc-calendar&token=YOUR_TOKEN&ip=" | curl -k -o ~/duckdns/duck.log -K -
EOF

chmod 700 duck.sh

# 6. 设置定时任务（每5分钟更新）
crontab -e
# 添加以下行：
*/5 * * * * ~/duckdns/duck.sh >/dev/null 2>&1
```

### 2.2 EU.ORG申请（备用）

1. 访问 https://nic.eu.org
2. 注册账号
3. 申请域名，如：cimc-calendar.eu.org
4. 等待审核（可能需要几天到几周）

### 2.3 Cloudflare配置

```bash
# 1. 访问 https://dash.cloudflare.com
# 2. 添加站点（使用DuckDNS或EU.ORG域名）
# 3. 按提示修改域名的DNS服务器
# 4. 等待DNS生效（可能需要24-48小时）
```

## 三、Cloudflare Tunnel配置

### 3.1 安装和认证

```bash
# 1. 安装cloudflared（如未安装）
brew install cloudflare-warp

# 2. 登录Cloudflare
cloudflared tunnel login

# 这会打开浏览器，选择要授权的域名
```

### 3.2 创建隧道

```bash
# 1. 创建隧道
cloudflared tunnel create zj-calendar

# 记录生成的隧道ID，如：xxxx-xxxx-xxxx-xxxx

# 2. 配置DNS路由
cloudflared tunnel route dns zj-calendar cimc-calendar.duckdns.org

# 3. 创建配置文件
mkdir -p ~/.cloudflared
cat > ~/.cloudflared/config.yml << 'EOF'
tunnel: YOUR_TUNNEL_ID
credentials-file: /Users/YOUR_USERNAME/.cloudflared/YOUR_TUNNEL_ID.json

ingress:
  - hostname: cimc-calendar.duckdns.org
    service: http://localhost:80
  - service: http_status:404
EOF
```

### 3.3 启动隧道

```bash
# 测试运行
cloudflared tunnel run zj-calendar

# 使用PM2管理（后台运行）
pm2 start cloudflared --name "cloudflared" -- tunnel run zj-calendar
pm2 save
```

## 四、Nginx配置

### 4.1 配置文件

```nginx
# /usr/local/etc/nginx/nginx.conf

user YOUR_USERNAME staff;
worker_processes auto;

events {
    worker_connections 1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;

    sendfile        on;
    keepalive_timeout  65;

    # Gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1000;

    # 上传文件大小限制
    client_max_body_size 20M;

    # 后端服务
    upstream backend {
        server 127.0.0.1:3000;
    }

    server {
        listen 80;
        server_name localhost cimc-calendar.duckdns.org *.duckdns.org;

        # 前端静态文件
        location / {
            root /Users/YOUR_USERNAME/zj-calendar/client/dist;
            index index.html;
            try_files $uri $uri/ /index.html;
        }

        # API代理
        location /api {
            proxy_pass http://backend;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_cache_bypass $http_upgrade;
        }

        # 上传文件
        location /uploads {
            alias /Users/YOUR_USERNAME/zj-calendar/server/uploads;
            expires 30d;
            add_header Cache-Control "public, immutable";
        }

        # 错误页面
        error_page 500 502 503 504 /50x.html;
        location = /50x.html {
            root html;
        }
    }
}
```

### 4.2 启动Nginx

```bash
# 测试配置
nginx -t

# 启动
brew services start nginx

# 或使用系统命令
sudo nginx

# 重新加载配置
nginx -s reload
```

## 五、应用部署

### 5.1 项目目录结构

```
/Users/YOUR_USERNAME/zj-calendar/
├── client/                 # 前端代码
│   ├── dist/              # 构建产物
│   └── ...
├── server/                 # 后端代码
│   ├── uploads/           # 上传文件
│   ├── prisma/
│   │   └── data.db        # SQLite数据库
│   └── ...
├── backups/               # 备份目录
└── logs/                  # 日志目录
```

### 5.2 后端部署

```bash
# 1. 克隆或上传代码到服务器
cd ~/zj-calendar/server

# 2. 安装依赖
pnpm install --prod

# 3. 生成Prisma客户端
pnpm prisma generate

# 4. 运行数据库迁移
pnpm prisma migrate deploy

# 5. 创建环境配置
cat > .env << 'EOF'
NODE_ENV=production
PORT=3000
DATABASE_URL="file:./prisma/data.db"
JWT_SECRET="your-super-secret-jwt-key-change-this"
JWT_EXPIRES_IN="7d"
EOF

# 6. 使用PM2启动
pm2 start npm --name "zj-calendar-api" -- run start:prod

# 7. 保存PM2配置
pm2 save

# 8. 设置开机自启
pm2 startup
# 按提示执行命令
```

### 5.3 前端部署

```bash
# 1. 进入前端目录
cd ~/zj-calendar/client

# 2. 安装依赖
pnpm install

# 3. 创建生产环境配置
cat > .env.production << 'EOF'
VITE_API_BASE_URL=/api
EOF

# 4. 构建
pnpm build

# 5. 构建产物在 dist/ 目录
# Nginx已配置指向此目录
```

### 5.4 PM2配置文件

```javascript
// ecosystem.config.js
module.exports = {
  apps: [
    {
      name: 'zj-calendar-api',
      script: 'npm',
      args: 'run start:prod',
      cwd: '/Users/YOUR_USERNAME/zj-calendar/server',
      env: {
        NODE_ENV: 'production',
        PORT: 3000
      },
      instances: 1,
      autorestart: true,
      watch: false,
      max_memory_restart: '500M',
      error_file: '/Users/YOUR_USERNAME/zj-calendar/logs/api-error.log',
      out_file: '/Users/YOUR_USERNAME/zj-calendar/logs/api-out.log',
      log_date_format: 'YYYY-MM-DD HH:mm:ss'
    },
    {
      name: 'cloudflared',
      script: 'cloudflared',
      args: 'tunnel run zj-calendar',
      autorestart: true,
      watch: false,
      error_file: '/Users/YOUR_USERNAME/zj-calendar/logs/tunnel-error.log',
      out_file: '/Users/YOUR_USERNAME/zj-calendar/logs/tunnel-out.log'
    }
  ]
};
```

## 六、数据备份

### 6.1 自动备份脚本

```bash
#!/bin/bash
# backup.sh - 每日备份脚本

BACKUP_DIR="/Users/YOUR_USERNAME/zj-calendar/backups"
DATE=$(date +%Y%m%d_%H%M%S)
KEEP_DAYS=30

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
cp /Users/YOUR_USERNAME/zj-calendar/server/prisma/data.db \
   $BACKUP_DIR/data_$DATE.db

# 备份上传文件（增量）
rsync -av /Users/YOUR_USERNAME/zj-calendar/server/uploads/ \
          $BACKUP_DIR/uploads_$DATE/

# 压缩备份
tar -czf $BACKUP_DIR/backup_$DATE.tar.gz \
    -C $BACKUP_DIR data_$DATE.db uploads_$DATE

# 删除临时文件
rm -rf $BACKUP_DIR/data_$DATE.db $BACKUP_DIR/uploads_$DATE

# 删除超过保留天数的备份
find $BACKUP_DIR -name "backup_*.tar.gz" -mtime +$KEEP_DAYS -delete

echo "Backup completed: backup_$DATE.tar.gz"
```

### 6.2 设置定时备份

```bash
# 编辑crontab
crontab -e

# 每天凌晨2点执行备份
0 2 * * * /Users/YOUR_USERNAME/zj-calendar/scripts/backup.sh >> /Users/YOUR_USERNAME/zj-calendar/logs/backup.log 2>&1
```

### 6.3 数据恢复

```bash
# 解压备份
cd /Users/YOUR_USERNAME/zj-calendar/backups
tar -xzf backup_20260301_020000.tar.gz

# 停止服务
pm2 stop zj-calendar-api

# 恢复数据库
cp data_20260301_020000.db ../server/prisma/data.db

# 恢复上传文件
rsync -av uploads_20260301_020000/ ../server/uploads/

# 重启服务
pm2 start zj-calendar-api
```

## 十一、更新部署流程

### 11.1 代码更新

```bash
# 1. 备份当前版本
cd ~/zj-calendar
tar -czf backup_before_update_$(date +%Y%m%d).tar.gz server/ client/

# 2. 拉取最新代码
cd ~/zj-calendar/server
git pull origin main
pnpm install --prod
pnpm prisma generate
pnpm prisma migrate deploy

cd ~/zj-calendar/client
git pull origin main
pnpm install
pnpm build

# 3. 重启服务
pm2 restart zj-calendar-api
nginx -s reload
```

### 11.2 回滚

```bash
# 解压备份
tar -xzf backup_before_update_YYYYMMDD.tar.gz

# 恢复并重启
pm2 restart zj-calendar-api
nginx -s reload
```
