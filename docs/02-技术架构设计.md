## 一、技术选型

### 1.1 选型原则

考虑到以下因素：
- 用户没有编程基础
- 后期运维成本最低
- 10-50人规模
- 自建服务器（Mac mini M4）
- **保留现有UI设计，确保用户体验一致性**

选择**技术成熟、社区活跃、易于维护**的技术栈。

### 4.1 用户认证流程

```

│                      用户认证流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【注册流程】                                                │
│                                                             │
│  用户输入邮箱 ──→ 检查邮箱是否已注册 ──→ 已注册: 提示登录   │
│         │                                    │              │
│         │ 未注册                              │              │
│         ▼                                    ▼              │
│  输入密码和确认密码 ─────────────────────────→│              │
│         │                                    │              │
│         ▼                                    │              │
│  选择安全问题并设置答案 ─────────────────────→│              │
│         │                                    │              │
│         ▼                                    │              │
│  保存用户信息到数据库 ───────────────────────→│              │
│         │                                    │              │
│         ▼                                    │              │
│  返回登录成功 + JWT Token ──────────────────→│              │
│                                                             │
│  【登录流程】                                                │
│                                                             │
│  用户输入邮箱+密码 ──→ 验证邮箱密码 ──→ 失败: 提示错误      │
│         │                                    │              │
│         │ 成功                                │              │
│         ▼                                    ▼              │
│  生成 JWT Token ─────────────────────────────→│              │
```

| SQL注入防护 | Prisma参数化查询 |
| XSS防护 | 前端输入过滤 + 后端输出编码 |

| 项目 | 方案 |
|------|------|
| 敏感数据加密 | 密码bcrypt加密 |
| 数据备份 | 每日自动备份SQLite文件 |
| 操作日志 | 记录关键操作 |
| 软删除 | 数据标记删除，30天后清除 |
