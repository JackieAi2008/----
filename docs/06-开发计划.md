## 五、验收标准

### 5.1 第一期验收标准

| 模块 | 验收标准 |
|------|----------|
| 用户认证 | 能正常注册、登录、找回密码 |
| 项目管理 | 能创建项目、邀请成员、设置可见性 |

| 深色模式 | 能切换深色/浅色模式 |

### 5.2 测试验收流程

```
1. 开发自测
   └── 开发者自行测试各功能

2. 小范围试用（3-5人）
   ├── 创建测试项目
   ├── 添加任务并指派
   ├── 测试日历视图
   ├── 测试通知功能
   └── 收集反馈

3. Bug修复
   └── 根据反馈修复问题

4. 正式发布
   └── 开放给所有用户
```

## 六、风险与应对

### 6.1 技术风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 内网穿透不稳定 | 中 | 高 | 配置备用穿透方案 |
| SQLite性能瓶颈 | 低 | 中 | 后期迁移PostgreSQL |
| PWA兼容性问题 | 中 | 低 | 降级为普通响应式 |

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 功能需求变更 | 中 | 中 | 预留缓冲时间 |
| 技术难点攻关 | 低 | 中 | 提前技术预研 |

### 8.2 推荐VS Code插件

| 插件 | 用途 |

| TypeScript Vue Plugin | TS支持 |

| Thunder Client | API测试 |

### 8.3 项目初始化命令

```bash
# 创建项目目录
mkdir zj-calendar && cd zj-calendar

# 初始化前端
pnpm create vite@latest client --template vue-ts
cd client
pnpm install
pnpm add element-plus vue-router pinia axios @vueuse/core
pnpm add -D sass

# 初始化后端
mkdir server && cd server
pnpm init
pnpm add express cors helmet morgan
pnpm add prisma @prisma/client
pnpm add bcryptjs jsonwebtoken express-validator
pnpm add -D typescript ts-node nodemon @types/*

# 初始化Prisma
pnpm prisma init
```

## 九、开发规范

### 9.1 代码规范

- 使用ESLint + Prettier统一代码风格
- TypeScript严格模式
- Vue 3 Composition API
- 组件命名：PascalCase
- 文件命名：kebab-case
- CSS使用BEM命名

### 9.2 Git提交规范

```
feat: 新功能
fix: Bug修复
docs: 文档更新
style: 代码格式调整
refactor: 重构
perf: 性能优化
test: 测试相关
chore: 构建/工具相关
```

### 9.3 分支管理

```
main        # 生产分支
├── dev     # 开发分支
│   ├── feature/xxx  # 功能分支
│   ├── fix/xxx      # 修复分支
│   └── refactor/xxx # 重构分支
```
