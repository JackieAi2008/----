> 版本：1.0.0
> 更新日期：2026年3月1日

## 一、数据库概述

### 1.1 数据库选型

| 项目 | 选择 | 说明 |
|------|------|------|
| 数据库类型 | SQLite | 轻量级，零配置，适合中小规模 |
| ORM工具 | Prisma | 现代化ORM，类型安全 |
| 数据库文件 | data.db | 单文件存储 |

### 1.2 数据库特点

- **零配置**：无需安装数据库服务
- **单文件**：备份只需复制一个文件
- **足够性能**：支撑50人并发使用
- **易于迁移**：后期可平滑迁移到PostgreSQL/MySQL

## 三、数据表详细设计

### 3.1 用户表 (User)

存储用户基本信息。

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | String | ✅ | UUID | 主键 |
| email | String | ✅ | - | 邮箱，唯一 |
| password | String | ✅ | - | 密码（bcrypt加密） |
| nickname | String | ✅ | - | 昵称 |
| avatar | String | ❌ | null | 头像URL |
| bio | String | ❌ | null | 个人简介 |
| isAdmin | Boolean | ✅ | false | 是否超级管理员 |
| isBanned | Boolean | ✅ | false | 是否被封禁 |
| createdAt | DateTime | ✅ | now() | 创建时间 |
| updatedAt | DateTime | ✅ | now() | 更新时间 |

**索引**：
- email (唯一索引)

### 3.2 安全答案表 (SecurityAnswer)

存储用户的安全问题和答案。

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | String | ✅ | UUID | 主键 |
| userId | String | ✅ | - | 用户ID（外键） |
| questionIndex | Int | ✅ | - | 问题序号（0-3） |
| answer | String | ✅ | - | 答案（加密存储） |
| createdAt | DateTime | ✅ | now() | 创建时间 |

**问题列表**：
- 0: 您的母亲姓名是什么？
- 1: 您的出生城市是哪里？
- 2: 您的第一所学校名称是什么？
- 3: 您最喜爱的电影是什么？

### 3.3 项目表 (Project)

存储项目信息。

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | String | ✅ | UUID | 主键 |
| name | String | ✅ | - | 项目名称 |
| description | String | ❌ | null | 项目描述 |
| cover | String | ❌ | null | 封面图片URL |
| visibility | Enum | ✅ | PRIVATE | 可见性（PUBLIC/PRIVATE） |
| ownerId | String | ✅ | - | 项目负责人ID（外键） |
| createdAt | DateTime | ✅ | now() | 创建时间 |
| updatedAt | DateTime | ✅ | now() | 更新时间 |
| deletedAt | DateTime | ❌ | null | 软删除时间 |

**索引**：
- ownerId (普通索引)
- visibility (普通索引)

### 3.4 项目成员表 (ProjectMember)

存储项目与用户的关系。

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | String | ✅ | UUID | 主键 |
| projectId | String | ✅ | - | 项目ID（外键） |
| userId | String | ✅ | - | 用户ID（外键） |
| role | Enum | ✅ | MEMBER | 角色（OWNER/MEMBER） |
| joinedAt | DateTime | ✅ | now() | 加入时间 |

**索引**：
- projectId, userId (联合唯一索引)

### 3.5 项目邀请表 (ProjectInvite)

存储项目邀请/申请记录。

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | String | ✅ | UUID | 主键 |
| projectId | String | ✅ | - | 项目ID（外键） |
| inviterId | String | ✅ | - | 邀请人ID（外键） |
| inviteeId | String | ✅ | - | 被邀请人ID（外键） |
| status | Enum | ✅ | PENDING | 状态（PENDING/APPROVED/REJECTED/EXPIRED） |
| expiresAt | DateTime | ✅ | 7天后 | 过期时间 |
| createdAt | DateTime | ✅ | now() | 创建时间 |

**索引**：
- projectId, inviteeId (联合索引)
- status (普通索引)

### 3.6 任务类别表 (TaskCategory)

存储任务类别。

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | String | ✅ | UUID | 主键 |
| projectId | String | ❌ | null | 项目ID（null表示系统默认） |
| name | String | ✅ | - | 类别名称 |
| color | String | ✅ | - | 颜色代码（如#FF5733） |
| isSystem | Boolean | ✅ | false | 是否系统默认 |

**系统默认类别**：

| 群团 | #3498DB |

## 四、Prisma Schema

```prisma
// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = "file:./data.db"
}

// 用户表
model User {
  id        String   @id @default(uuid())
  email     String   @unique
  password  String
  nickname  String
  avatar    String?
  bio       String?
  isAdmin   Boolean  @default(false)
  isBanned  Boolean  @default(false)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  // 关联
  securityAnswers SecurityAnswer[]
  ownedProjects   Project           @relation("ProjectOwner", fields: [ownerId], references: [id])
  projectMembers  ProjectMember[]
  sentInvites     ProjectInvite[]   @relation("Inviter")
  receivedInvites ProjectInvite[]   @relation("Invitee")
  assignedTasks   Task[]            @relation("TaskAssignee")
  createdTasks    Task[]            @relation("TaskCreator")
  taskCollaborators TaskCollaborator[]
  comments        Comment[]
  attachments     Attachment[]
  notifications   Notification[]
  auditLogs       AuditLog[]
}

// 安全答案表
model SecurityAnswer {
  id           String   @id @default(uuid())
  userId       String
  questionIndex Int
  answer       String
  createdAt    DateTime @default(now())

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([userId, questionIndex])
}

// 项目表
model Project {
  id          String     @id @default(uuid())
  name        String
  description String?
  cover       String?
  visibility  Visibility @default(PRIVATE)
  ownerId     String
  createdAt   DateTime   @default(now())
  updatedAt   DateTime   @updatedAt
  deletedAt   DateTime?

  // 关联
  owner      User            @relation("ProjectOwner", fields: [ownerId], references: [id])
  members    ProjectMember[]
  invites    ProjectInvite[]
  categories TaskCategory[]
  tasks      Task[]

  @@index([visibility])
}

// 项目成员表
model ProjectMember {
  id        String   @id @default(uuid())
  projectId String
  userId    String
  role      ProjectRole @default(MEMBER)
  joinedAt  DateTime @default(now())

  project Project @relation(fields: [projectId], references: [id], onDelete: Cascade)
  user    User    @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([projectId, userId])
}

// 项目邀请表
model ProjectInvite {
  id        String     @id @default(uuid())
  projectId String
  inviterId String
  inviteeId String
  status    InviteStatus @default(PENDING)
  expiresAt DateTime
  createdAt DateTime   @default(now())

  project Project @relation(fields: [projectId], references: [id], onDelete: Cascade)
  inviter User    @relation("Inviter", fields: [inviterId], references: [id], onDelete: Cascade)
  invitee User    @relation("Invitee", fields: [inviteeId], references: [id], onDelete: Cascade)

  @@index([status])
}

// 任务类别表
model TaskCategory {
  id        String  @id @default(uuid())
  projectId String?
  name      String
  color     String
  isSystem  Boolean @default(false)

  project Project @relation(fields: [projectId], references: [id], onDelete: Cascade)
  tasks   Task[]
}

// 任务表
model Task {
  id          String       @id @default(uuid())
  projectId   String
  title       String
  description String?
  startDate   DateTime?
  dueDate     DateTime
  categoryId  String?
  assigneeId  String
  priority    Priority     @default(MEDIUM)
  status      TaskStatus   @default(TODO)
  deliverable String?
  tags        String?      // JSON array
  reminder    Reminder?
  repeat      Repeat?
  creatorId   String
  createdAt   DateTime     @default(now())
  updatedAt   DateTime     @updatedAt
  deletedAt   DateTime?

  project       Project           @relation(fields: [projectId], references: [id], onDelete: Cascade)
  category      TaskCategory?     @relation(fields: [categoryId], references: [id], onDelete: SetNull)
  assignee      User              @relation("TaskAssignee", fields: [assigneeId], references: [id], onDelete: Cascade)
  creator       User              @relation("TaskCreator", fields: [creatorId], references: [id], onDelete: Cascade)
  collaborators TaskCollaborator[]
  comments      Comment[]
  attachments   Attachment[]

  @@index([dueDate])
  @@index([status])
}

// 任务协作者表
model TaskCollaborator {
  id     String @id @default(uuid())
  taskId String
  userId String

  task Task @relation(fields: [taskId], references: [id], onDelete: Cascade)
  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([taskId, userId])
}
```

### 5.2 备份策略

| 备份类型 | 频率 | 保留时间 |
|----------|------|----------|
| 全量备份 | 每天 | 30天 |
| 增量备份 | 每小时 | 7天 |

**备份脚本**：
```bash
#!/bin/bash
# 每日备份脚本
DATE=$(date +%Y%m%d_%H%M%S)
cp /path/to/data.db /path/to/backup/data_$DATE.db
# 保留最近30天的备份
find /path/to/backup -name "data_*.db" -mtime +30 -delete
```

### 5.3 迁移到其他数据库

如需迁移到PostgreSQL或MySQL：

1. 修改 `schema.prisma` 中的数据库配置
2. 运行 `prisma migrate dev` 生成迁移
3. 使用数据库工具导出SQLite数据并导入新数据库
