# UI 增强设计方案

> 日期：2026-03-06
> 状态：待实现

## 概述

本设计包含三个 UI 增强功能，提升用户体验和操作效率。

---

## 功能一：概览页面卡片展开

### 需求描述

概览页面的四个统计卡片（今日待办、逾期任务、本周任务、参与项目）需要支持点击展开，显示具体任务/项目内容。

### 交互设计

**默认状态：**
- 卡片显示统计数字和图标
- 四个卡片排列成 2x2 网格（桌面端）或单列（移动端）

**展开状态：**
- 点击卡片展开显示内容列表（手风琴效果）
- 同一时间只能展开一个卡片
- 再次点击或点击其他卡片可切换

**内容展示：**
- 固定显示 3 条内容
- 每条内容：类别色点 + 标题 + 辅助信息
- 底部显示「查看全部 N 条」

### 各卡片内容

| 卡片 | 内容 | 辅助信息 |
|------|------|----------|
| 今日待办 | 任务标题 | 优先级标签 |
| 逾期任务 | 任务标题 | 逾期天数 |
| 本周任务 | 任务标题 | 截止日期 |
| 参与项目 | 项目名称 | 任务数量 |

---

## 功能二：日历点击创建任务

### 需求描述

在日历的各个视图中，点击空白区域应该能够快速创建该日期/时间的任务。

### 各视图交互行为

| 视图 | 点击位置 | 弹出行为 |
|------|----------|----------|
| 年视图 | 任意日期数字 | 打开新建任务对话框，截止日期设为该日 18:00 |
| 月视图 | 日期格子空白区域 | 打开新建任务对话框，截止日期设为该日 18:00 |
| 周视图 | 时间格子（1小时） | 打开新建任务对话框，开始/截止时间设为该小时 |
| 日视图 | 时间轴区域 | 打开新建任务对话框，开始/截止时间设为该小时 |
| 甘特图 | 任务条 | 跳转任务详情页（不创建任务） |

### 任务表单预设值

年/月视图点击：截止时间设为点击日期 18:00
周/日视图点击：开始时间=点击小时，截止时间=点击小时+1

---

## 功能三：任务表单 - 项目选择器增强

### 需求描述

在任务表单的「所属项目」选择器旁添加新建项目入口，支持快速创建项目并选择成员。

### 交互设计

- 项目选择器右侧添加 [+] 按钮
- 点击弹出项目快速创建对话框
- 创建成功后自动选中新项目

### 项目快速创建对话框字段

- 项目名称（必填）
- 项目描述（可选）
- 可见性（公开/私密）
- 项目成员（支持搜索选择）

### 角色权限

| 角色 | 权限 |
|------|------|
| 项目负责人 | 编辑项目信息、添加/移除成员、删除项目 |
| 普通成员 | 查看项目、创建/编辑自己的任务 |

---

## 实现优先级

1. P0：日历点击创建（最常用功能）
2. P1：项目选择器增强（提升效率）
3. P2：概览卡片展开（锦上添花）
