# 部门成员管理功能设计

## 概述

在部门管理页面内增加完整的成员管理功能，包括查看成员列表、添加成员、移除成员、更换管理员。

## 页面布局

采用左右分栏布局：

```
┌──────────────────────────────────────────────────────────────┐
│ 统计卡片区域（保持现有）                                        │
├──────────────────────────────────────────────────────────────┤
│ 标题 + 创建部门按钮                                            │
├────────────────────┬─────────────────────────────────────────┤
│   部门列表          │         部门详情面板                     │
│   (约 1/3 宽度)     │         (约 2/3 宽度)                   │
│                    │                                         │
│   点击选中部门       │   部门信息 + 成员列表表格                │
│                    │                                         │
└────────────────────┴─────────────────────────────────────────┘
```

## 组件设计

### 1. 成员列表表格

| 列 | 内容 | 宽度 | 说明 |
|---|------|------|------|
| 头像 | 用户头像 | 48px | 圆形头像，无头像显示默认图标 |
| 昵称 | 用户昵称 | 自适应 | 主要标识 |
| 邮箱 | 用户邮箱 | 自适应 | 次要标识 |
| 角色 | 管理员/成员 | 80px | 标签样式区分 |
| 操作 | 操作按钮 | 100px | 设为管理员、移除 |

**角色标签样式：**
- 管理员：蓝色标签 `bg-blue-100 text-blue-700`
- 成员：灰色标签 `bg-gray-100 text-gray-600`

**操作按钮：**
- 成员行：显示「设为管理员」「移除」按钮
- 管理员行：仅显示「移除」

### 2. 添加成员弹窗

**功能：**
- 默认显示所有「未分配部门」的用户列表
- 支持搜索所有用户（昵称/邮箱）
- 搜索结果中显示用户当前所属部门
- 支持多选批量添加
- 添加已属于其他部门的用户时，自动从原部门移出

### 3. 移除成员弹窗

**功能：**
- 显示该成员负责的任务数量和列表
- 要求选择任务接收人进行转移
- 可选择「不转移任务」直接移除
- 确认后：移除部门归属 + 转移任务（如选择）

### 4. 更换管理员弹窗

**功能：**
- 显示当前管理员和即将变更的管理员
- 确认后更新部门 adminId
- 原管理员自动降为普通成员

## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/departments/:id/members` | GET | 获取部门成员列表 |
| `/departments/:id/members` | POST | 添加成员到部门（支持批量） |
| `/departments/:id/members/:userId` | DELETE | 移除成员（可选 transferTo 参数转移任务） |
| `/departments/:id/admin` | PUT | 更换部门管理员 |
| `/users?unassigned=true` | GET | 获取未分配部门的用户 |
| `/users/search?q=keyword` | GET | 搜索用户（返回部门和角色信息） |

## 数据模型

使用现有的 `User` 和 `Department` 模型，无需修改 schema。

**关键关系：**
- `User.departmentId` → 用户所属部门
- `Department.adminId` → 部门管理员

## 实现范围

1. **前端**

   - 新增成员列表表格组件

   - 新增移除成员弹窗
   - 新增更换管理员弹窗

2. **后端**
   - 新增成员管理相关路由
   - 实现任务转移逻辑
